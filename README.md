# 🏦 SecureTrust Bank — Loan Approval Prediction System

An end-to-end Machine Learning project that predicts whether a loan
application will be **Approved (Y)** or **Rejected (N)** based on
applicant demographics, income, and loan details. Includes EDA, model
training & comparison, a Flask web app with a banking-style Bootstrap 5
UI, and Vercel deployment configuration.

## 1. Project Overview

**Problem statement:** Predict loan approval status (`Loan_Status`)
based on:

| Feature | Description |
|---|---|
| Gender | Male / Female |
| Married | Applicant's marital status |
| Dependents | Number of dependents (0, 1, 2, 3+) |
| Education | Graduate / Not Graduate |
| Self_Employed | Yes / No |
| ApplicantIncome | Monthly income of applicant |
| CoapplicantIncome | Monthly income of co-applicant |
| LoanAmount | Loan amount requested (in thousands) |
| Loan_Amount_Term | Term of the loan (in days) |
| Credit_History | 1 = good credit history, 0 = poor/none |
| Property_Area | Urban / Semiurban / Rural |

**Target:** `Loan_Status` — `Y` (Approved) or `N` (Rejected)

### Pipeline summary

1. **EDA** — shape, dtypes, missing values, target distribution,
   correlation heatmap, and visual breakdowns (`notebooks/EDA.ipynb`
   and plots auto-saved to `static/images/` by `train_model.py`).
2. **Cleaning** — the raw dataset contains real-world noise (typos like
   `Mle`/`Fmale`, inconsistent codes like `Y`/`N` vs `Yes`/`No`, outlier
   sentinels such as `LoanAmount = 9999` or `Loan_Amount_Term = 999`,
   and invalid `Credit_History` codes). All of this is normalized in
   `preprocessing.py::clean_raw_values()`.
3. **Imputation** — numerical columns filled with **median**,
   categorical columns filled with **mode**.
4. **Feature engineering** — `TotalIncome`, log-transformed income/loan
   amounts, `Loan_Income_Ratio`, approximate `EMI`, and `Balance_Income`.
5. **Encoding & scaling** — Label Encoding for categoricals,
   `StandardScaler` for numeric features.
6. **Model training & comparison** — Logistic Regression, Decision Tree,
   and Random Forest are trained and compared on Accuracy, Precision,
   Recall, F1 Score, and Confusion Matrix. The model with the best F1
   score is automatically selected and persisted.
7. **Flask app** — a form-based UI that runs the *exact same*
   preprocessing pipeline (`preprocess_single_record`) before calling
   `model.predict()`.

## 2. Installation

### Prerequisites
- Python 3.10+
- pip

### Steps

```bash
# 1. Clone / unzip the project
cd loan-approval-system

# 2. (Recommended) create a virtual environment
```

## 3. Training the Model

Run the full training pipeline. This performs EDA (printing summaries
and saving plots to `static/images/`), cleans the data, engineers
features, trains all three models, prints a comparison table, and
saves `model.pkl`, `encoder.pkl`, and `scaler.pkl` to `model/`.

```bash
python train_model.py
```

**Sample output:**

```
              Model  Accuracy  Precision  Recall  F1 Score
Logistic Regression     0.662     0.6627  0.9985    0.7966
      Random Forest     0.660     0.6626  0.9925    0.7947
      Decision Tree     0.632     0.6637  0.9020    0.7647

Best model selected: Logistic Regression
```

> The best model is selected automatically based on **F1 Score**. If
> you retrain with different data/parameters, the saved model and the
> printed comparison table will reflect the new results.

To explore the data interactively, open `notebooks/EDA.ipynb` in
Jupyter:

```bash
jupyter notebook notebooks/EDA.ipynb
```

## 4. Running the Flask App Locally

After training (so that `model/*.pkl` files exist), start the app:

```bash
python app.py
```

Then open your browser at **http://127.0.0.1:5000**.

- Fill in the loan application form.
- Click **"Check Loan Eligibility"**.
- The result panel will show **Loan Approved ✅** or
  **Loan Rejected ❌**, along with an approval probability bar.

A `/health` endpoint is also available for quick status checks
(`{"status": "ok", "model_loaded": true}`).

## 5. Deployment (Vercel)

This project includes a `vercel.json` configured for Flask via the
`@vercel/python` runtime.

### Steps

1. **Important:** Train the model locally first so `model/*.pkl` exist
   — Vercel will package these files with your deployment.
   ```bash
   python train_model.py
   ```

2. Install the Vercel CLI (if you don't have it):
   ```bash
   npm install -g vercel
   ```

3. From the project root, deploy:
   ```bash
   vercel
   ```
   Follow the prompts to link/create a project. For production:
   ```bash
   vercel --prod
   ```

4. Vercel will build using `requirements.txt` and route all traffic to
   `app.py` (Flask's `app` WSGI object), serving `/static/*` directly.

### Notes on Vercel deployment
- Vercel's serverless functions are stateless and ephemeral — the
  app **loads `model.pkl`, `encoder.pkl`, and `scaler.pkl` from disk on
  each cold start**, which is fast since they are small pickle files.
- Set `app.run(debug=False)` (already the default in production via
  the WSGI entry point) — `debug=True` should only be used locally.
- If your model files are large, consider Git LFS or external storage
  (e.g., S3) and load them at runtime instead of bundling.

## 7. Future Improvements

- **Hyperparameter tuning** via `GridSearchCV` / `RandomizedSearchCV`
  for all three models.
- **Additional models** — XGBoost, LightGBM, or a stacked ensemble.
- **SHAP / feature importance** explanations shown alongside each
  prediction for transparency.
- **User authentication** and application history/storage (e.g., via a
  database) for returning applicants.
- **API endpoint** (`/api/predict`) returning JSON for integration with
  other systems or a mobile app.
- **Class imbalance handling** (SMOTE / class weighting) to improve
  recall on the minority (Rejected) class.
- **Input range hints / tooltips** in the UI based on dataset
  statistics to guide users.
- **Automated testing** (unit tests for `preprocessing.py` and
  integration tests for `app.py` routes).
